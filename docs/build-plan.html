Critical Path Analysis

W1 ──▶ W3 ──▶ W5
  • W1 → W3: Realtime client must emit transcript + tool events into the state machine before agent simulation can react.
  • W3 → W5: Demo orchestration drives state transitions (dispatch agents, trigger blocker, escalate). Without state, nothing animates on cue.
  • Slack zero: any slip on the critical path pushes the demo finale. If W1 isn't shipping by T+25, fall back to "pre-recorded voice" mode — Realtime mute, scripted audio playback. Visual demo still works.
  • W2 must finish by T+25: no UI = no demo surface. This is the second most painful path.
  • W4 slips are survivable: lose ArtifactPreview last; Moodboard alone still demos the "voice → visual" beat.

Ruthless cuts (post-MVP — do NOT build today)

  • Real Codex spawning — agents are simulated (timer-driven state advance). The voice/UI experience is what we're selling.
  • Session rotation (60-min cap) — demo is < 60 min.
  • Compaction — irrelevant in demo window.
  • Harness persistence to disk — keep in memory; rule save still flashes Canvas.
  • FormField (mic permission) Canvas — assume permission granted at start.
  • Code preview Canvas — skip.
  • Diagram Canvas (Mermaid) — skip.
  • Onboarding welcome sequence — first launch goes straight to dormant.
  • Session resume prompt — always fresh.
  • Hotkey conflict detection — assume ⌘⇧Space available.
  • Auto-DND / macOS Focus integration — skip.
  • Multi-monitor / responsive — primary display only.
  • Reduced motion / accessibility polish — phase 2.
